nextjs가 이미 있어서 일부 데이터를 서버사이드에서 불러와서 랜더링 하는 것이 가능해서 사실 따로 알아보고 있지는 않았는데
react conf 에서 우연히 소개하는 설명을 보고 내가 일부 착각한 게 있어서 정리해보았다.

---

서버컴포넌트는 서버에서 실행이되고 결과물이 시리얼화되어서 그대로 전달된다.
이후 일반 컴포넌트가 hydration과정을 마친 뒤에 가상 돔에 서버컴포넌트 결과물이 병합된다.

따라서 서버 컴포넌트에서 사용한 js 라이브러리는 네트워크 요청상 전달되는 번들에서 제외되고
기존 nextjs는 서버에서 네트워크 요청은 가능했지만 이런 부분이 불가능 했기 때문에
서버 컴포넌트를 효과적으로 활용할 수록 성능상 이점이 커진다고 할 수 있겠다.
